PART I HADOOP INFRASTRUCTURE

Hadoop is a platform that provides both distributed storage (HDFS) and distributed computation (MapReduce) capabilities. It parallelizes and executes programs on a cluster of machines with a “master - slave” mode. It provides high scalability and fault-tolerance.

(Image Source )

Components:

  • HBase: NoSQL database. hosting very large tables (billions of rows * millions of columns). Random access, real time read and write.
  • Pig: High level language to express data analysis programs. Easier to program than MapReduce and can optimization. Example: Load 'student' USINg PigStorage() AS (name: chararry, age: int, gpa:float);
  • Hive: Querying structured data with SQL-like syntax from RDBMS with MapReduce execution. Example: SELECT a.foo from invites a WHERE a.date='<DATE>'
  • Mahout: Predictive analytics tool for machine learning tasks.

Limitations:

  • HDFS is not for random reads over small files. It’s for batch read / write over large files.
  • Compatibility issues for different versions of Hadoop, Pig etc.
  • Security: no storage or wire-level encryption in Hadoop.
  • Master node single point of failure. But improved in Hadoop v.2.x.


PART II SPARK

  • Motivation:
    • Aim at solving the limitation of MapReduce: a lot of Disk I/O -> slow and needs a lot of time.
    • Cost of memory is declining log-linearly per year. -> keeping more data in memory is feasible.

Components:

  • Spark SQL: SQL, structured data processing
  • Spark Streaming: Real time streaming
  • MLlib: machine learning
  • GraphX: graph processing

Comparison with Hadoop

Hadoop Spark
Storage Disk only In-memory / On disk
Operations Map Reduce Map Reduce Join Sample
Execution Batch Batch, interactive, streaming
Programming environment Java (other APIs available) Scala (other APIs available)

Example:

  • For logistic regression, Hadoop needs 80 seconds but Spark only needs 0.96 seconds;
  • For K-means clustering, Hadoop needs 121 seconds but Spark only needs 4 seconds.

So, Hadoop is more used in offline storage, such as data warehousing. Spark is more used for large data computation.

RDD & DataFrame

  • RDD: low level, partitioned collections of objects
    • High level transformation
    • Typed data (structured or semi-structured)
    • when debugging, DAG visualizations within Spark UI references RDDs
  • DataFrame: high level, built on top of RDD
    • Need low-level transformations
    • Unstructured or schema-less data (e.g. media, text)
RDD DataFrames
Schema Doesn’t have Have a schema
Storage More Less memory (saved in its own binary format, 1/4 storage)
Time More Less time ( apply an optimizer in its API functionality)
Flexibility More Less
