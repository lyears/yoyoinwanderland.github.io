Data Science Interview Questions from Giants

Posted on 2017-04-06
Key notes from Heinz Data Science Job Hunting Team

This post is to answer a collection of data science interview questions posted on LearnDataSci. Special thanks to my CMU friends Xiaoyu, Shuting, Qianyi.

General Questions

Apple

  • Suppose you’re given millions of users that each have hundreds of transactions and these millions of transactions are for tens of thousands of products. How would you group the users together in meaningful segments?

    This depends on business objective. If the management aim to know different degrees of engagement, then sales amount, purchase frequency, purchase range are valuable inputs to cluster on. If the team wants to group “neighbours” in terms of product preferences, the transaction records could be a large one hot encoding matrix to be clustered on.

  • Describe a project you’ve worked on and how it made a difference.

  • How would you approach a categorical feature with high-cardinality?

    Categorical feature should always be dealt with one hot encoding. When it has high cardinality, first thing to do is to understand the goal of the task, what categorical feature represents and its distribution. If this is a prediction task and we want to improve accuracy, feature reduction such as PCA might be a good choice; while if it’s a causality analysis task, feature selection algorithms such as lasso (logistic) regression is more appropriate. In terms of feature meaning, if, say, the feature represents age range and it’s divided into many little ranges, some combinations could be made. If we observe the distribution with bar chart and we find some of the categories take the majority of all the records, then maybe we could combine all the rest into one category.

    Additional answers from ref:

    • When using Scikit-learn, DictEncoder helps to tranform categorical features to one hot encoding sparse matrix thus saving memory - as the matrix is supposed to be very sparse.

    • Tree-based algorithms can deal with categorical variables. For high-cardinality categorical feature, random forest is recommended. It bootstraps for features and records, thus it removes some noisy / unuseful features.

  • What would you do to summarize a Twitter feed?

    A twitter feed always states either an event, or a mood. So we can either do a POS tagging with NLTK, and then select either adjective plus noun, noun plus noun if not found then adjective only and make these as summation; or we can use named entity analytics (HMM) using NLTK and sentiement analysis using Stanford NLP tool.

    ​

  • What are the steps for wrangling and cleaning data before applying machine learning algorithms? ref

    For numerical data, the normal things we do include:

    • Data joining. Sometimes we are provided with multiple sources of files we need to combine them;
    • Manipulate missing data: there are several ways to deal with missing data: we can either delete rows containing missing data if it won’t lose important info; or we can use zero/ average fill it; or we can use a regression to estimate missing data.
    • Deal with outliers: firstly observe the meaning of the feature and if the value all make sense, if no then remove it; if yes, smooth with regression line, or use clustering; then remove outliers

    • Data normalization is always useful in any prediction tasks

      For text data:

    • Standarization: Stem (remove the ending) or lemmatize (go back to the original form) it

    • one hot encoding: transform to term frequency matrix or tf-idf matrix

  • How do you measure distance between data points?

    For numerical data, we can use Minkowski distance. When P=1 it’s mahanttan distance and P=2 it’s euclidean distance. So as P increases it penalizes more on big differences (outliers for linear regression, misclassified items for classification).

    For text data, we can use cosine similarity for tokenized matrix.

  • Define variance.

    Variance is the disperse from the mean.

  • Describe the differences between and use cases for box plots and histograms.

    For box plots we can see the ranges as well as the percentiles with just one line, so it’s good for distribution comparison between different categories. With histogram we can see the distributions (i.e. the shape) of the data.

  • How would you find an anomaly in a distribution ? Ref

    There are two kinds of anomaly detection:

    • Novelty detection: to detect meaningful anomaly patterns in a clean dataset. This is normally used in fraud detection in banking/ insurance/ telecom industries. One of the common ways is to use an elliptic envolope with assumption that the data comes from certain “shape” of distribution such as Gaussian.
    • Outlier detection: to identify outliers from a noisy dataset. There are several ways, like visualization with one or two dimensional data, fitting an elliptic envolope, or an isolation forest for high dimensional data.

  • How would you go about investigating if a certain trend in a distribution is due to an anomaly?

  • How would you estimate the impact Uber has on traffic and driving conditions?

    A/B testing on regions with/ without uber driving. Useful metrics: average time waiting for taxi, traffic conjunction duration in peak hours etc.​

  • What metrics would you consider using to track if Uber’s paid advertising strategy to acquire new customers actually works? How would you then approach figuring out an ideal customer acquisition cost?

    Metric: Clickthrough rate and conversion rate. Normlly customer acquisition cost is calculated using $ Total market campaign / No of customers acquired$, but it should be noted that the as total market campain increases, the curve won’t be a straight line, instead it should be diminishing. The ideal customer acquisition cost can be calculated before the turning point.

Machine Learning

Google

  • Why do you use feature selection?

    Three reasons:

    • Due to Ocam’s Razor, the simpler the better
    • Normally the number of rows should be less than the number of features
    • Multicolinearity: We assume the features are independent from each other but in reality this is not the case.

  • What is the effect on the coefficients of logistic regression if two predictors are highly correlated? What are the confidence intervals of the coefficients?

    This causes four problems: unstable coefficients, interpretation problems for coefficients, causes R^2 to increase, and inflated P-value on correlated coefficients.

    Confidence interval is, with large number of experiments, coefficients will fall into the certain range. It’s not where true value of coefficients will reside.

  • What’s the difference between Gaussian Mixture Model and K-Means?

    Gaussian mixture model is a soft clustering method while k-means is a hard clustering method.

    Gaussian is based on EM - the probabilities resides in certain cluster. K-means is based on distance.

  • How do you pick k for K-Means?

    Plot SSE (Sum of square error) onto graph and choose the point after the steepest step.

  • How do you know when Gaussian Mixture Model is applicable?

    K-means produces less accurate results than Gaussian mixture model. So if the resources are enough, kmeans is a good first step to determine initlization points.

  • Assuming a clustering model’s labels are known, how do you evaluate the performance of the model?

    Cross entropy loss. It measures how “pure” each cluster is.

  • What is a neural network?

    It is a classification method that model brain neuras to solve complex classification problems. Back propagation is a “learn from mistake” algorithm for neural network training. Back propagation, in its nature, is gradient descent. It compares the output with the label, and “propagate” the error back onto the previous layers and adjust the weights accordingly.

  • Explain the Bias-Variance Tradeoff

    $MSE = bias + variance$

  • How do you deal with unbalanced binary classification?

    Oversampling the samller class and undersample the big class. Similarly, we could give more weights to smaller class when calculating objective losses. In scikit-learn, there is a parameter class_weight and it could be set as “balanced”.

  • What’s the difference between L1 and L2 regularization?

    L1 regularization gives sparse estimation by giving a Lapace prior for coefficients. It will force most of coefficients to be zero.
    L2 regularization will retain all predictors with Gaussian prior for coefficients. Some of the coefficients get larger and some get smaller. It’s easier to compute.
